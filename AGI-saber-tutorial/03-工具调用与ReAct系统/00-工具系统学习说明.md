# 00-工具系统学习说明

## 1. 这个目录怎么学

这个模块按“方法调用链”学习工具系统，不按概念零散记。

推荐从完整教学版开始：

```text
01-完整教学版/
```

学习顺序是：

```text
processInternal
→ ChatRouter
→ ToolService
→ ToolModeHandler
→ PreferenceFiller
→ execute.apply
→ ReActLoop
→ Planner
→ TaskGraph
→ GraphRuntime
→ ChatGenerator
```

## 3. 这一模块要掌握什么

学完后要能讲清楚：

```text
1. 工具从哪里注册进来
2. Tool / ToolParam / ToolCallResult 的区别
3. 单工具模式怎么从 query 决策出 ToolCallResult
4. PreferenceFiller.fill 怎么补参数，为什么不覆盖用户显式参数
5. tool.getExecute().apply(params) 到底执行了什么
6. 为什么单工具不够，需要 ReAct
7. Planner 怎么生成 Node
8. TaskGraph 怎么表达 depends_on
9. GraphRuntime 怎么并发、竞速、重试和收集结果
10. ChatGenerator 为什么要把工具结果再交给 LLM 总结
```

## 4. 和其他模块的关系

```text
主链路教学：
  告诉你请求怎么进入 UnifiedAgentService，并在 switch(mode) 中进入 tool/react。

记忆系统：
  给工具系统提供 memPrefix 和 PreferenceMemory。

工具调用与 ReAct 系统：
  解释 tool/react 两条分支内部到底怎么执行。
```
