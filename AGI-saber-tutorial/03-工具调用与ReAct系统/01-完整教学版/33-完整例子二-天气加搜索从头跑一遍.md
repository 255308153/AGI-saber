# 33-完整例子二：天气加搜索从头跑一遍

## 1. 这条 query 走完 ReAct 完整链路

```text
query = "查一下上海天气，并搜索小雨出门建议"
```

它触发 `needReAct`（两个维度：天气 + 搜索），进入 ReAct 模式。从 `planner.planGraph` 到 `ChatGenerator.generate`，每一步用对象状态快照展示，标注"详见第 X 章"。

## 2. 输入

```text
query = "查一下上海天气，并搜索小雨出门建议"
req.explicit = false
tools = {get_time, get_weather, search_web, rag_search}
memPrefix = "用户偏好 城市：上海"
```

## 3. 走查

### Step 1: decideMode → react

```text
详见第 2, 4 章

decideMode("查一下上海天气，并搜索小雨出门建议", false, ...)
  → needReAct:
      天气: ✓ → count=1
      查/搜索: ✓ → count=2
      sum/汇总: 无
      count=2 >= 2 → true
  → return "react"
```

### Step 2: Planner.planGraph → 3 个 Node

```text
详见第 18, 19 章

planGraph("查一下上海天气，并搜索小雨出门建议", ts, memPrefix)

LLM 输出：
[
  {"id":"n1","type":"tool","tool":"get_weather",
   "params":{"city":"上海"},"reason":"查询上海天气",
   "depends_on":[],"race_group":""},
  {"id":"n2","type":"tool","tool":"search_web",
   "params":{"query":"小雨出门建议"},"reason":"搜索雨天出行建议",
   "depends_on":["n1"],"race_group":"search"},
  {"id":"n3","type":"tool","tool":"rag_search",
   "params":{"query":"小雨出门建议"},"reason":"从知识库搜索建议",
   "depends_on":["n1"],"race_group":"search"}
]

格式1解析成功 → 3个Node

planNodes:
  n1: {toolName:"get_weather", params:{city:"上海"}, dependsOn:[], raceGroup:""}
  n2: {toolName:"search_web",  params:{query:"小雨出门建议"}, dependsOn:["n1"], raceGroup:"search"}
  n3: {toolName:"rag_search", params:{query:"小雨出门建议"}, dependsOn:["n1"], raceGroup:"search"}
```

### Step 3: new TaskGraph + validate

```text
详见第 21, 22, 23 章

new TaskGraph(planNodes):
  nodes:    {n1→Node, n2→Node, n3→Node}
  adjList:  {n1→[n2,n3], n2→[], n3→[]}
  inDegree: {n1→0, n2→1, n3→1}

validate():
  检查悬空依赖: n2→"n1"存在 ✓, n3→"n1"存在 ✓
  topologicalLevels():
    Round1: ready=[n1] → L0=[n1]
            n1→n2入度减1→0, n1→n3入度减1→0
    Round2: ready=[n2,n3] → L1=[n2,n3]
    processed=3 == 3 ✓
  → levels = [[n1], [n2, n3]]
```

### Step 4: GraphRuntime.execute

```text
详见第 24 章

Level 0: ["n1"]
  groupByRace → [RaceGroup("", ["n1"])]
  空组 → runSingle("n1")
    doExecuteNode("n1", null)                ← 第28章
      setStatus → RUNNING
      invoke(n1)                              ← 第29章
        → TOOL → get_weather.apply({city:"上海"})
        → "上海：小雨 20°C"
      setStatus → DONE
      result = "上海：小雨 20°C"

Level 1: ["n2", "n3"]
  groupByRace → [RaceGroup("search", ["n2","n3"])]
  竞速组 → runRace(pool, g)                  ← 第27章
    n2 线程: search_web → Tavily搜索(较慢)
    n3 线程: rag_search → 本地向量检索(较快)
    n3 先完成 → winnerFound CAS true
    → n3: DONE, result="雨天建议：带好雨具..."
    → n2: SKIPPED (n2线程被interrupt)

buildResult:                                  ← 第31章
  observations:
    "[get_weather] 上海：小雨 20°C"
    "[rag_search] 雨天建议：带好雨具..."
  nodeResults:
    n1→{DONE, "上海：小雨 20°C", ""}
    n2→{SKIPPED, "", ""}
    n3→{DONE, "雨天建议：带好雨具...", ""}
```

## 4. 单工具 vs ReAct 对比

| 维度 | 单工具 (第32章) | ReAct (本章) |
|---|---|---|
| 入口方法 | `ToolModeHandler.run` | `ReActLoop.runStream` |
| 工具决策 | `ToolService.decide` (规则) | `Planner.planGraph` (LLM) |
| 执行方式 | 顺序，一次 | DAG，按层并行+竞速 |
| 工具数量 | 1 个 | 多个 |
| LLM 总结 | 内联 `llm.chat` | `ChatGenerator.generate` |
| 偏好补参 | `PreferenceFiller.fill` | 无（Planner 直接填好） |
| 失败处理 | 返回错误消息 | 重试 3 次 / 单个节点 FAILED |
| 竞速 | 无 | 有 |
