# 32-完整例子一：上海天气怎么样

## 1. 这条 query 走完单工具完整链路

本章用一条具体的 query——`"上海天气怎么样？"`——追踪从 `processInternal` 到 `llm.chat` 的每一步。不重复贴源码，用对象状态快照展示数据流。每一步标注"详见第 X 章"。

## 2. 输入

```text
query = "上海天气怎么样？"
req.explicit = false
req.useRag = false
req.selectedTools = []
ragLoaded = true
pref 中存有："城市" → "杭州"     ← 注意：偏好城市和 query 城市不同
tools = {get_time, get_weather, search_web, rag_search}
```

## 3. 走查

### Step 1: processInternal 开始

```text
详见第 1 章

cancelled = false
resp.mode = "chat"（默认值）
stm.add("user", "上海天气怎么样？")
异步 + 同步偏好抽取（无新偏好产出）
memPrefix = "用户偏好 城市：杭州\n\n【相关记忆】...\n"
histMsgs = [{role:"user", content:"上海天气怎么样？"}]
```

### Step 2: decideMode

```text
详见第 2 章

decideMode("上海天气怎么样？", false, false, [], true)
  → explicit=false → 进入自动规则
  → needReAct?
      时间/几点: 无 → 0
      天气: 上海天气怎么样？ 含"天气" → 1
      总结/汇总: 无 → 1
      查/搜索: 无 → 1
      count=1 < 2 → false
  → needTool?
      "天气"命中 → true
  → return "tool"

mode = "tool"
toolset = tools (全量)
```

### Step 3: switch("tool") → ToolModeHandler.run

```text
详见第 12 章

进入 ToolModeHandler.run(resp, "上海天气怎么样？", toolset, memPrefix, histMsgs)
```

### Step 4: ToolService.decide

```text
详见第 13 章

decide("上海天气怎么样？", toolset)

q = "上海天气怎么样？"
分支1 (时间): q.contains("几点")||q.contains("时间") → false
分支2 (天气): q.contains("天气") → true
             + tools.containsKey("get_weather") → true
             遍历城市: "上海"命中 → params.put("city", "上海")
             return ToolCallResult("get_weather", {city: "上海"})

tc = {
  toolName: "get_weather"
  params: {city: "上海"}       ← 用户显式说了
  toolResult: null
}
```

### Step 5: ts.get

```text
ts.get("get_weather")
  → 返回启动时 createGetWeatherTool 创建的 Tool 对象
  → tool != null ✓
```

### Step 6: PreferenceFiller.fill

```text
详见第 14 章

fill(tc, pref):

pref.getData() = {"城市": "杭州"}    ← 用户偏好城市是杭州

遍历 PREF_TO_PARAM:
  "城市" → [city, location, location_name]
  prefVal = "杭州"（非空）
  遍历候选参数名:
    paramName="city"
      v = tc.params.get("city") → "上海"   ← 非空！
      → 不写入！

  → tc.params 不变：{city: "上海"}

关键结论：用户显式说了"上海"，偏好"杭州"被忽略。
```

### Step 7: tool.getExecute().apply

```text
详见第 15 章

tool.getExecute().apply({city: "上海"})

进入 createGetWeatherTool 的 execute lambda:
  city = "上海"（不为 null，跳过默认"北京"）
  WEATHER_DB.get("上海") → "小雨 20°C"
  return "上海：小雨 20°C"

tc.setToolResult("上海：小雨 20°C")

tc = {
  toolName: "get_weather"
  params: {city: "上海"}
  toolResult: "上海：小雨 20°C"   ← 写入
}
```

### Step 8: LLM 总结

## 4. 全链路状态变化一览

```text

  ↓ decideMode

```

## 5. 关键设计决策回顾

| 决策 | 体现的地方 | 为什么 |
|---|---|---|
| 关键词触发工具 | `needTool` 匹配"天气" | 低成本快速路由，不调 LLM |
| 规则提取参数 | `decide` 遍历城市列表 | 从 query 中直接取，不需要 LLM 推理 |
| 偏好不覆盖显式输入 | `fill` 检查 `v==null \|\| empty` | 尊重用户当前意图 |
| 工具结果不直接返回 | LLM 总结 | 原始数据 ≠ 用户可读回答 |
