# 09-最容易说错的点

## 1. 召回不是异步

错误：

```text
长期/图记忆召回是异步的。
```

正确：

```text
召回是回答前同步执行。因为召回结果要进入 system prompt，直接影响当前回答质量。
```

---

## 2. 长期记忆不是只有回答后才写

错误：

```text
长期记忆回答后才写。
```

正确：

```text
query 后异步偏好抽取会写长期记忆。
answer 后 MemoryWriter 也会写长期记忆。
```

---

## 3. MemoryWriter 只是其中一条写入路径

错误：

```text
MemoryWriter = 长期记忆全部写入逻辑。
```

正确：

```text
MemoryWriter 负责回答后从 assistant answer 分类抽取长期记忆。
query 后的偏好长期记忆由 runAsyncPreferenceExtraction 写。
```

---

## 4. importance 不是 score

错误：

```text
importance 就是召回分数。
```

正确：

```text
importance 是写入时的重要性。
score 是召回时临时算的综合分。
score = sim * 0.7 + importance * 0.3。
```

---

## 5. category 固定，tags 不固定

错误：

```text
tag 一共有五种。
```

正确：

```text
category 有五种：identity、preference、tool_failure、policy、general。
tags 是自由标签，不是固定枚举。
```

---

## 6. requiredTags 是 AND

错误：

```text
requiredTags 里有一个命中就可以。
```

正确：

```text
当前 requiredTags 是 AND。必须全部包含才通过。
```

---

## 7. 过滤不是越细越好

错误：

```text
过滤条件越多，召回越准。
```

正确：

```text
过滤太细会漏召回。多意图 query 要拆分子 query，不要把多个意图的 tag 全塞进 requiredTags。
```

---

## 8. FOLLOWS 不是因果

错误：

```text
FOLLOWS 表示前一条导致后一条。
```

正确：

```text
FOLLOWS 只表示写入顺序。
```

---

## 9. BELONGS_TO 当前不是自动主路径

错误：

```text
图记忆会自动创建 BELONGS_TO。
```

正确：

```text
当前 GraphMemory.storeClassified 自动创建的是 FOLLOWS 和 SIMILAR_TO。BELONGS_TO 不是自动写入主路径。
```

---

## 10. consolidation 不是召回

错误：

```text
consolidation 是召回的一部分。
```

正确：

```text
consolidation 是后台整理，负责衰减、去重、合并、过期删除和同步 DB/图。
```

---

## 11. 添加去重不能替代 consolidation

错误：

```text
添加记忆时已经去重，所以不需要 consolidation。
```

正确：

```text
添加去重只比较当前新记忆。consolidation 会定期全量两两比较，还会做衰减、合并、过期清理。
```

---

## 12. 三份存储不是强一致

错误：

```text
内存、PostgreSQL、Neo4j 是事务强一致。
```

正确：

```text
当前实现是最终一致。内存先整理，DB 和 Neo4j 根据 ConsolidationResult 后续同步，Neo4j 删除还是异步。
```

---

## 13. 图保护不严格保护内存

错误：

```text
图中心度保护能保证节点不从内存删除。
```

正确：

```text
当前 graphAwareConsolidate 先 ltm.consolidate 改内存，再过滤 deleteFromDB，所以主要保护 DB 和 Neo4j，不严格保护当前 JVM 内存。
```

---

## 14. 面试前最后背这段

```text
这个记忆系统里，召回是同步读，写入和整理大多异步。回答前会把偏好和长期/图召回结果拼成 memPrefix，把短期历史转成 histMsgs。长期记忆有两条写入路径：query 后偏好抽取写入，answer 后 MemoryWriter 写入。图记忆是在长期记忆 seed 基础上扩展 Neo4j 邻居。consolidation 后台做去重、合并、过期删除，并通过 ConsolidationResult 同步 PostgreSQL 和 Neo4j。当前是最终一致，不是强一致。
```
