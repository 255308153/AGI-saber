# 04-长期记忆写入时机-两条异步路径

## 1. 一句话结论

长期记忆不是只有回答后才写。

准确说：

```text
query 后：异步偏好抽取会写长期记忆。
answer 后：MemoryWriter 会写长期记忆。
```

两条都是异步，所以都不保证当前轮稳定生效。

---

## 2. 第一条：query 后偏好写入长期记忆

源码入口：

```java
runAsyncPreferenceExtraction(query);
```

它会启动新线程：

```java
new Thread(() -> {
    Map<String, String> kvs = llm.extractPreferences(query);
    ...
}, "preference-extract").start();
```

流程：

```text
用户 query
-> LLM 抽取偏好 Map<String,String>
-> 写 PreferenceMemory
-> 写 preferences 表
-> 转成自然语言事实
-> llm.embed(content)
-> storeMemory(content, 0.8, emb)
-> 保存 long_term_memory
-> syncMemoryPGID(pgId)
```

例子：

```text
query = "我叫小李，我喜欢 Java 逐行解释"

抽取：
姓名 = 小李
回答偏好 = Java 逐行解释

转长期记忆：
content = "用户回答偏好: Java 逐行解释"
importance = 0.8
```

---

## 3. 第二条：answer 后 MemoryWriter 写长期记忆

源码入口：

```java
memoryWriter.writeAfterReply(query, resp.getAnswer());
```

它也会启动新线程：

```java
new Thread(() -> {
    writeNow(query, answer);
}, "memory-writer").start();
```

流程：

```text
assistant answer
-> LLM 分类抽取 JSON items
-> category / content / tags
-> 根据 category 计算 importance
-> llm.embed(content)
-> ltm.storeClassified 或 graphMem.storeClassified
-> 保存 PostgreSQL
-> syncLastItemPGID(pgId)
```

例子：

```text
Classified{
  category = "general",
  content = "用户正在学习图记忆召回链路",
  tags = ["图记忆", "记忆系统"]
}
```

---

## 4. 两条路径对比

| 对比项 | query 后偏好抽取 | answer 后 MemoryWriter |
|---|---|---|
| 触发时间 | 用户问题进来后 | LLM 回答结束后 |
| 是否异步 | 是 | 是 |
| 抽取来源 | 用户 query | assistant answer |
| 主要内容 | 偏好、身份 | 分类长期事实 |
| 是否保证当前轮生效 | 不保证 | 不可能影响已经生成的回答 |

---

## 5. 最容易说错

错误说法：

```text
长期记忆回答后才写。
```

正确说法：

```text
长期记忆有两条异步写入路径。偏好类长期记忆在 query 后就启动写入；回答内容类长期记忆在 answer 后由 MemoryWriter 写入。
```

---

## 6. 面试怎么说

```text
长期记忆写入分两条异步路径。第一条是用户 query 进来后启动 LLM 偏好抽取，把偏好转成自然语言事实写入长期记忆；第二条是回答结束后 MemoryWriter 从 answer 中分类抽取长期事实写入长期记忆。两条路径都是异步，所以本轮回答稳定依赖的是已有长期记忆召回，而不是本轮刚写入的新记忆。
```
