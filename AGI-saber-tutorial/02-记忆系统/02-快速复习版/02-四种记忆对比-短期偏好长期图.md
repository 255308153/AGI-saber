# 02-四种记忆对比-短期偏好长期图

## 1. 一句话结论

四种记忆不是重复设计，它们解决的问题不同：

```text
短期记忆管最近上下文。
偏好记忆管用户稳定习惯。
长期记忆管可语义召回的事实。
图记忆管记忆之间的关系。
```

---

## 2. 短期记忆

对象：

```text
ConversationMessage{
  role = "user" 或 "assistant",
  content = "消息正文",
  timestamp = "时间"
}
```

作用：

```text
让 LLM 知道最近几轮聊了什么。
```

例子：

```text
user: 图记忆怎么召回？
assistant: 先召回 seed，再扩展 Neo4j 邻居。
```

进入哪里：

```text
histMsgs
```

---

## 3. 偏好记忆

对象：

```text
PreferenceMemory.data = {
  "姓名": "小李",
  "回答偏好": "喜欢 Java 逐行解释",
  "默认城市": "上海"
}
```

作用：

```text
让回答更符合用户稳定偏好。
```

进入哪里：

```text
memPrefix 里的【用户偏好】
```

---

## 4. 长期记忆

对象：

```text
MemoryItem{
  id = 101,
  content = "用户正在学习 AGI-saber 记忆系统",
  importance = 0.8,
  embedding = [...],
  category = "general",
  tags = ["记忆系统", "面试"]
}
```

作用：

```text
通过 embedding 语义召回，把相关事实带回 prompt。
```

进入哪里：

```text
memPrefix 里的【相关记忆】
```

---

## 5. 图记忆

对象：

```text
(:Memory {mem_id: 101, content: "...", importance: 0.8})

(100)-[:FOLLOWS]->(101)
(101)-[:SIMILAR_TO {weight: 0.83}]->(108)
```

作用：

```text
长期记忆先召回 seed，然后从 Neo4j 扩展邻居，补充相关上下文。
```

---

## 6. 最容易混淆

```text
histMsgs 是短期记忆，不是长期记忆。
memPrefix 是偏好 + 相关长期/图记忆，不是聊天历史。
PreferenceMemory 是 KV，LongTermMemory 是 MemoryItem。
图记忆不是单独替代长期记忆，它依赖长期记忆 ID。
```

---

## 7. 面试怎么说

```text
我把记忆分成四层。短期记忆保存最近对话，负责上下文连续性；偏好记忆保存用户稳定习惯，直接拼到 system prompt；长期记忆保存可语义召回的自然语言事实，通过 embedding 和 score 找相关内容；图记忆在长期记忆基础上把 MemoryItem 映射到 Neo4j 节点，通过 FOLLOWS 和 SIMILAR_TO 表达顺序和相似关系。
```
